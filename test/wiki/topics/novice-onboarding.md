## Summary

The fixture sources agree that novice onboarding should hide implementation details, ask at most three useful questions, and point users to a readable start page.

## Evidence

- `raw/docs/product_meeting_notes.docx` records the decision to ask at most three onboarding questions.
- `raw/notes/meeting_scribbles.txt` says not to expose internal folder words in the first response.
- `raw/notes/messy_research_notes.md` says a novice should not need to know what `schema/` means.

## Empty-folder rule

If there is almost no source material, offer a starter pack and then ask for three seed memories. Do not create many empty placeholder pages.

## Uncertainty

The fixture does not validate which starter pack is best for each domain; it only tests that starter packs prevent hollow scaffolding.
